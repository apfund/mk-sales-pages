## Logo System

### Primary Logo
**Wordmark: Finance-able** (with hyphen)

```
F i n a n c e - a b l e
```

**Logo Construction:**
- Primary typeface: Custom display font (or Inter Extra Bold as interim)
- The hyphen serves as a visual separator and brand signature element
- Lowercase "a" in "able" emphasizes accessibility and capability
- Capital "F" establishes authority

### Logo Variations

**1. Primary Full Color**
- Use on white or light backgrounds
- Default for most applications

**2. Reversed (White)**
- Use on dark backgrounds or photography
- Maintains full logo structure

**3. Monochrome**
- Single color applications
- Use primary blue (#0A2647) or black

**4. Simplified Mark**
- "F-a" lettermark for app icons, favicons
- Square format: 512x512px minimum
- The hyphen becomes the central design element

### Logo Clear Space
Minimum clear space equals the height of the lowercase "a" on all sides. Never place text, graphics, or other elements within this zone.

### Minimum Sizes
- Digital: 120px width minimum
- Print: 1 inch width minimum
- Favicon/App Icon: 32px minimum (use simplified mark)

### Logo Don'ts
❌ Don't use multiple brand name spellings (Finance|able, Financeable)  
❌ Don't stretch or distort the logo  
❌ Don't add effects (shadows, gradients, outlines)  
❌ Don't rotate the logo  
❌ Don't place on busy backgrounds without sufficient contrast  
❌ Don't recreate or modify the logo

### Color Usage Guidelines

**60-30-10 Rule**
- 60% Neutral (Mocha Background, whites, grays)
- 30% Primary (Deep Finance Blue)
- 10% Accent (Growth Green, Warm Coral, Premium Gold)

**Background Combinations**
✓ **Approved:**
- Charcoal text on Mocha Background
- White text on Deep Finance Blue
- Deep Finance Blue text on white
- Growth Green on white (large text only)

❌ **Never Use:**
- Growth Green text on Deep Finance Blue
- Warm Coral text on Deep Finance Blue
- Light Gray text on Mocha Background (fails contrast)
- Slate Gray text on Light Gray (fails contrast)

**Color Accessibility Matrix**

| Background | Text Colors (WCAG AA Pass) |
|------------|---------------------------|
| Mocha Background (#FAF9F6) | Charcoal (#1F2937), Deep Finance Blue (#0A2647), Slate Gray (#64748B) |
| White (#FFFFFF) | Charcoal (#1F2937), Deep Finance Blue (#0A2647), Growth Green (#10B981, large text) |
| Deep Finance Blue (#0A2647) | White (#FFFFFF), Mocha Background (#FAF9F6) |
| Growth Green (#10B981) | White (#FFFFFF), Charcoal (#1F2937) |

Always verify contrast ratios at [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) before implementation.

## Typography System

### Primary Typeface: Inter

**Why Inter?**
- Designed specifically for screens and UI
- Excellent readability at small sizes
- Variable font support for performance
- Extensive weight range (100-900)
- Open source, free for commercial use

**Download:** [rsms.me/inter](https://rsms.me/inter/)

### Headline Fonts

**Extra Bold (800)**
- Hero headlines
- Page titles
- Major section headers
- Size: 48-72px (desktop), 32-48px (mobile)

**Bold (700)**
- Subheadings
- Card titles
- Navigation items
- Size: 24-36px (desktop), 20-28px (mobile)

**Semibold (600)**
- Small headers
- Emphasized text
- Labels
- Size: 16-20px

### Body Text Fonts

**Regular (400)**
- Primary body text
- Paragraphs
- Lists
- Size: 16-18px (optimal for reading)
- Line height: 1.6-1.8 (optimal for comprehension)

**Medium (500)**
- Emphasized body text
- Sidebar content
- Quotes

### Supporting Text

**Light (300)**
- Large display text (when paired with small caps or all caps)
- Minimal use—only for specific design moments

### Type Scale

**Desktop Scale:**
```
H1: 56px / 700 weight / 1.1 line-height / -0.02em letter-spacing
H2: 40px / 700 weight / 1.2 line-height / -0.01em letter-spacing
H3: 32px / 600 weight / 1.3 line-height / normal letter-spacing
H4: 24px / 600 weight / 1.4 line-height / normal letter-spacing
H5: 20px / 600 weight / 1.5 line-height / normal letter-spacing
H6: 18px / 600 weight / 1.5 line-height / normal letter-spacing

Body Large: 18px / 400 weight / 1.7 line-height
Body: 16px / 400 weight / 1.6 line-height
Body Small: 14px / 400 weight / 1.6 line-height
Caption: 12px / 500 weight / 1.5 line-height / uppercase / 0.05em letter-spacing
```

**Mobile Scale (Adjust by 0.8-0.9x):**
```
H1: 40px / 700 weight / 1.1 line-height
H2: 32px / 700 weight / 1.2 line-height
H3: 28px / 600 weight / 1.3 line-height
Body: 16px / 400 weight / 1.6 line-height (maintain base size)
```

### Optional: Custom Serif Headline Font

**For Premium Differentiation (Phase 2)**

Consider custom serif or high-quality serif like:
- **Freight Display** (sophisticated, warm)
- **Tiempos Headline** (modern, confident)
- **GT Sectra** (elegant, distinctive)

**Usage if implemented:**
- Large headlines only (H1, H2)
- Hero sections
- Premium course branding
- Print materials

**Budget:** $500-2,000 for licensing

### Typography Best Practices

**Hierarchy**
- Never use more than 3 font sizes per section
- Maintain consistent heading sizes across pages
- Use weight and color to create hierarchy, not just size

**Readability**
- Maximum line length: 70-80 characters (optimal: 50-60)
- Minimum body text size: 16px (never smaller)
- Never use all caps for paragraphs
- Use sentence case for buttons and CTAs

**Performance**
- Load Inter as variable font (single file, all weights)
- Subset fonts to include only Latin characters
- Use `font-display: swap` to prevent invisible text

**Accessibility**
- Maintain WCAG AA contrast ratios (4.5:1 minimum)
- Allow users to resize text up to 200%
- Never use font size alone to convey meaning
- Ensure text remains readable when zoomed

## Spacing and Grid System

### 8-Point Grid System

All spacing uses multiples of 8:
```
4px   (0.25rem) - Tight spacing (icon padding, small gaps)
8px   (0.5rem)  - Minimum spacing unit
16px  (1rem)    - Base spacing (element padding, small margins)
24px  (1.5rem)  - Medium spacing (card padding, section gaps)
32px  (2rem)    - Large spacing (section margins)
48px  (3rem)    - Extra large (major section breaks)
64px  (4rem)    - Section dividers
96px  (6rem)    - Major page sections (desktop only)
```

### Container Widths

**Maximum Content Width:** 1280px  
**Standard Content Width:** 1120px  
**Reading Width:** 720px (for long-form content)  
**Sidebar Width:** 320px

### Responsive Breakpoints

```
Mobile: 320px - 767px
Tablet: 768px - 1023px
Desktop: 1024px - 1439px
Large Desktop: 1440px+
```

### Grid Columns

**Desktop (1024px+):** 12 columns  
**Tablet (768-1023px):** 8 columns  
**Mobile (<768px):** 4 columns

**Gutter:** 24px (32px on large desktop)

**Icon System: Lucide Icons**

- Modern, clean line icons
- Consistent 24x24px grid
- Extensive library (1000+ icons)
- Open source, MIT license
- Available as React components, SVGs, or icon font

**Download:** [lucide.dev](https://lucide.dev/)

**Icon Sizing**
```
Extra Small: 16px (inline with text)
Small: 20px (form inputs, small buttons)

```

**Icon Colors**
```

Secondary: Slate Gray (#64748B)
Success: Growth Green (#10B981)
```

**Icon Usage Guidelines**
- Always pair with accessible text label (visible or aria-label)
- Maintain consistent stroke width (2px standard)
- Use filled icons sparingly for emphasis
- Ensure 24x24px minimum touch target on mobile

**Common Icons Used**
- Navigation: Menu, X (close), ChevronDown, ChevronRight

- Communication: Mail, MessageCircle, Bell, User

### Illustrations

**Illustration Style: Custom Line Art**

**Characteristics:**
- Clean, minimal line drawings
- 2-3px stroke weight
- Limited color palette (Deep Finance Blue, Growth Green, Warm Coral)
- Isometric or flat perspective (not 3D)
- Human figures: Simple, diverse, professional attire
- Financial concepts: Abstract geometric representations

**Use Cases:**
- Empty states ("No courses yet")
- Hero sections (desktop only)
- Feature explanations
- Onboarding flows
- Error pages (404, 500)

**Budget:** $200-500 per custom illustration  
**Alternative:** Use illustration libraries like Undraw or Storyset temporarily, customize colors to brand palette

**Illustration Dos:**
✓ Keep backgrounds simple or transparent  
✓ Align to 8px grid  
✓ Use brand colors exclusively  
✓ Maintain consistent stroke weight across all illustrations  
✓ Include diverse representation in human figures

**Illustration Don'ts:**
❌ Don't use multiple styles across the site  
❌ Don't use clip art or dated stock illustrations  
❌ Don't make illustrations overly complex or detailed  
❌ Don't use illustrations on mobile if they don't add value

### Photography

**Photography Style: Authentic Practitioner**

**Preferred Aesthetics:**
- Natural lighting, not overly stylized
- Real work environments (trading floors, offices, laptops)
- Diverse professionals (age, gender, ethnicity)
- Focused on actions: modeling in Excel, analyzing reports, collaborating
- Close-ups showing hands on keyboards, pens on paper
- Behind-the-scenes teaching moments

**Avoid:**
- Generic stock photos with cheesy smiles
- Overly posed corporate headshots
- Perfect, sterile environments
- Single demographic representation
- Handshake photos (cliché)

**Photography Treatment:**
```
Aspect Ratios:
  Hero Images: 21:9 or 16:9
  Course Thumbnails: 16:9
  Team Photos: 4:3 or 1:1 (square)
  Blog Headers: 16:9

Overlays:
  Dark Overlay: rgba(10, 38, 71, 0.6) for text legibility
```

## Motion and Animation

### Animation Philosophy

**Purpose Over Polish**
Every animation must serve a functional purpose:
- Guide attention to important elements
- Provide feedback for user actions
- Indicate loading or processing states
- Celebrate achievements and milestones
- Clarify spatial relationships

**Performance First**
- Target 60fps consistently
- Use CSS transforms and opacity (GPU-accelerated)
- Avoid animating width, height, top, left (triggers layout)
- Implement "reduce motion" media query

### Animation Principles

**Duration Guidelines**
```
Micro-interactions: 100-200ms (button hover, input focus)
Transitions: 200-300ms (page elements, cards)
Modals/Overlays: 200-300ms (enter), 150-200ms (exit)
Celebrations: 400-600ms (success animations)
Complex Sequences: 800-1200ms (onboarding flows)
```

**Easing Functions**
```
ease-out: Starts fast, ends slow
  Use for: Elements entering viewport, user-initiated actions
  Cubic-bezier: cubic-bezier(0, 0, 0.2, 1)

ease-in: Starts slow, ends fast
  Use for: Elements exiting viewport, dismissals
  Cubic-bezier: cubic-bezier(0.4, 0, 1, 1)

ease-in-out: Slow start and end, fast middle
  Use for: Elements moving between positions
  Cubic-bezier: cubic-bezier(0.4, 0, 0.2, 1)

spring (bouncy): Natural, playful movement
  Use for: Celebrations, achievements, delightful moments
  Cubic-bezier: cubic-bezier(0.34, 1.56, 0.64, 1)
```

### Accessibility: Reduced Motion

**Critical Implementation**
```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

**What to Maintain with Reduced Motion:**
- Instant state changes (no animation)
- Color transitions (opacity fades OK, very subtle)
- Focus indicators (no transition delay)
- Loading states (show immediately, no spinner animation)

**What to Disable with Reduced Motion:**
- Parallax effects
- Auto-playing carousels
- Looping animations
- Zoom/scale effects
- Complex entrance animations

## Voice and Tone

### Brand Voice

Finance-able speaks as a **confident practitioner, not an ivory tower academic**. We're the mentor who's been in the trenches—building models at 2am, pitching to demanding PE partners, analyzing companies in real time.

**Voice Characteristics:**
- **Confident:** We know what works because we've done it
- **Conversational:** Professional but approachable, like talking to a colleague
- **Efficient:** No fluff. Every word serves a purpose
- **Empowering:** "You can do this. Here's how."
- **Authentic:** Real experiences, real results, real challenges

**We Sound Like:**
- A senior colleague explaining a complex model over coffee
- A mentor who wants you to succeed and will tell you the truth
- An industry veteran who's seen it all but still remembers being new

**We Don't Sound Like:**
- A stuffy professor reading from a textbook
- An aggressive salesperson pushing a product
- A guru making unrealistic promises
- A corporate robot using jargon to sound smart

### Tone Variations by Context

**Homepage/Marketing:**
- Confident and outcome-focused
- "Land your dream role at Goldman Sachs, KKR, or Blackstone"
- Direct, clear value propositions
- Empowering: "You're capable. We'll show you how."

**Course Content/Educational:**
- Patient and instructional
- "Let's walk through this step-by-step"
- Encouraging: "This is complex, but you're getting it"
- Practical: "In real life, you'd approach this by..."

**Community/Support:**
- Warm and encouraging
- "We're in this together"
- Empathetic: "I remember struggling with DCFs too"
- Supportive: "That's a great question—here's how to think about it"

**Transactional (Checkout, Confirmations):**
- Clear and reassuring
- "You're all set. Here's what happens next."
- Efficient with necessary information
- Friendly but focused on action

### Writing Guidelines

**Use:**
- Active voice: "Build financial models" not "Financial models are built"
- Contractions: "You'll learn" not "You will learn"
- Second person: "You" not "Students" or "Learners"
- Specific numbers: "$200,000 starting salary" not "high salary"
- Real examples: "As I learned pitching to Bain Capital..." not generic cases

**Avoid:**
- Jargon without explanation (unless teaching that term)
- Excessive adverbs: "really," "very," "extremely"
- Passive constructions: "It is recommended" → "We recommend"
- Corporate speak: "leverage," "synergy," "optimize" (unless teaching finance terms)
- Hyperbole: "best," "perfect," "ultimate" (unless backed by data)

**Sentence Structure:**
- Vary length: mix short punchy sentences with longer explanatory ones
- Average 15-20 words per sentence
- One idea per sentence
- Use bullet points for lists (like this!)

**Paragraph Guidelines:**
- Keep paragraphs 2-4 sentences
- One main idea per paragraph
- Use subheadings every 2-3 paragraphs for scanability
- White space enhances comprehension

## Accessibility Standards

### WCAG 2.2 Level AA Compliance

Finance-able commits to **WCAG 2.2 Level AA accessibility** as the minimum standard across all digital properties. This is both an ethical imperative (democratizing access) and a legal requirement.

**Compliance Deadline:** June 28, 2025 (European Accessibility Act)  
**US ADA Compliance:** April 24, 2026 (Title II)

### Four Core Principles (POUR)

**1. Perceivable**

Users must be able to perceive all information:

✓ **Text Alternatives**
- All images have descriptive alt text
- Decorative images use empty alt (`alt=""`)
- Complex images (charts, diagrams) have extended descriptions
- Icons paired with visible or aria-label text

✓ **Color Contrast**
- Normal text: 4.5:1 minimum contrast ratio
- Large text (18px+): 3.0:1 minimum contrast ratio
- All color combinations tested and documented in Color System
- Information never conveyed by color alone (use icons, text, patterns)

✓ **Adaptable Content**
- Logical reading order in HTML
- Semantic HTML headings (H1 → H2 → H3, no skipping)
- Form labels programmatically associated with inputs
- Data tables use proper markup (<th>, scope attributes)

✓ **Distinguishable**
- Text resizable up to 200% without loss of functionality
- No content relies on sound alone
- Background audio can be paused or volume controlled
- Visual focus indicators always visible (never removed)

**2. Operable**

Users must be able to navigate and interact:

✓ **Keyboard Accessible**
- All functionality available via keyboard
- Tab order follows logical reading order
- No keyboard traps (user can tab away from all elements)
- Skip links provided ("Skip to main content")

✓ **Enough Time**
- No time limits on content (or extend option provided)
- Auto-updating content can be paused/hidden
- Session timeouts warned with option to extend

✓ **Seizure Prevention**
- No content flashes more than 3 times per second
- Large flashing areas avoided entirely

✓ **Navigable**
- Every page has descriptive title
- Focus order makes sense
- Link text describes destination ("View course details" not "click here")
- Multiple ways to find pages (nav, search, sitemap)
- Current page indicated in navigation

✓ **Input Modalities (WCAG 2.2)**
- Touch targets minimum 24x24 CSS pixels
- Pointer cancellation (can change mind before releasing)
- Drag operations have alternative (copy/paste, buttons)

**3. Understandable**

Content and interface must be clear:

✓ **Readable**
- Language specified in HTML (<html lang="en">)
- Unusual words explained or defined
- Abbreviations defined on first use
- Reading level appropriate for content (aim for grade 9-10)

✓ **Predictable**
- Navigation consistent across pages
- Components function consistently
- Automatic updates don't change context unexpectedly
- Focus never jumps without user action

✓ **Input Assistance**
- Form errors identified and described
- Labels or instructions provided for inputs
- Error suggestions offered when detected
- Critical actions (delete, submit) require confirmation

✓ **Consistent Help (WCAG 2.2)**
- Help mechanisms in consistent location across pages
- Contact information, chat, FAQs always accessible

**4. Robust**

Content must work with assistive technologies:

✓ **Compatible**
- Valid HTML markup (no unclosed tags, duplicate IDs)
- ARIA labels used correctly (not overused)
- Status messages announced to screen readers
- Custom components have proper ARIA roles
- Tested with NVDA (Windows), JAWS (Windows), VoiceOver (Mac/iOS)

### WCAG 2.2 New Success Criteria (Added)

**Focus Appearance (Enhanced)**
- Focus indicator minimum 2px thick
- High contrast ratio (3:1 against background)
- Fully visible, not obscured by other content

**Focus Not Obscured**
- Focused element not hidden by sticky headers/footers
- At least partially visible when receiving focus

**Dragging Movements**
- All drag operations have single-pointer alternative
- Example: File upload has both drag-and-drop and browse button

**Target Size (Minimum)**
- Interactive elements minimum 24x24 CSS pixels
- Exception: Inline links in paragraphs

**Consistent Help**
- Help mechanisms in same relative order across pages
- Contact, chat, FAQ links in consistent locations

**Redundant Entry**
- Information entered previously auto-filled or retrieved
- Example: Shipping = billing address option

**Accessible Authentication**
- No cognitive function tests (memorize password, solve puzzle)
- Allow password managers, biometric authentication
- Email verification links vs. typing codes

### Testing and Validation

**Automated Testing Tools:**
- **axe DevTools** (Chrome/Firefox extension) – Free, comprehensive
- **WAVE** (WebAIM) – Visual feedback overlay
- **Lighthouse** (Chrome DevTools) – Built-in, accessibility score
- **Pa11y** – Command line testing, CI/CD integration

**Manual Testing Required:**
- Keyboard navigation (Tab, Shift+Tab, Enter, Space, Arrow keys)
- Screen reader testing (NVDA, JAWS, VoiceOver)
- Color contrast verification (Contrast Ratio tool)
- Zoom to 200% and verify functionality
- Mobile screen reader (TalkBack Android, VoiceOver iOS)

**User Testing:**
- Test with actual users with disabilities
- Conduct usability testing with assistive tech users
- Gather feedback on pain points
- Iterate based on real-world usage

**Financial Data Tables:**

```html
  <caption>Quarterly Revenue by Region</caption>
  <thead>
    <tr>
      <th scope="col">Quarter</th>
      <th scope="col">North America</th>
      <th scope="col">Europe</th>
      <th scope="col">Asia</th>

      <td>$1.2M</td>
      <td>$800K</td>
      <td>$600K</td>
```

**Interactive Charts:**
- Provide text-based data table alternative
- Keyboard navigation to data points
- Announce data values to screen readers
- High contrast mode option

**Video Content:**
- Closed captions for all videos (manually reviewed, not auto-generated)
- Audio descriptions for visual-only content
- Transcripts available as downloadable text
- Video controls accessible via keyboard
- No auto-play (or provide pause button immediately)

```html
<label for="email">Email Address <span aria-label="required">*</span></label>

  id="email" 
  name="email"
  aria-required="true"
  aria-describedby="email-help email-error"
  aria-invalid="false"
>
<div id="email-help">We'll never share your email.</div>
<div id="email-error" role="alert" aria-live="polite"></div>
```

**Financial Calculators:**
- All inputs have clear labels
- Error messages specific and actionable
- Results announced to screen readers
- Units clearly stated ($, %, years)
- Keyboard accessible for all interactions

**For Innovation:**
- Testing new ideas in controlled experiments
- Exploring future design directions
- Creating special campaign experiences

**For Context:**
- Platform-specific requirements (e.g., social media dimensions)
- Accessibility improvements (always prioritized)
- User testing reveals better approaches

**Documentation:**
When breaking rules, document:
1. What rule was broken
2. Why it was necessary
3. Results/learnings from the deviation
4. Whether to incorporate into future guidelines

## Brand Applications (continued)

### Email Marketing (continued)

**Template Structure:**
- Header: Logo (left), navigation links (right)
- Hero: Full-width image or background color with headline

- CTA: Primary button, centered, generous padding
- Footer: Unsubscribe, address, social icons

**Typography:**

- Links: Deep Finance Blue, underlined on hover

**Best Practices:**
- Mobile-first design (70%+ users on mobile)
- Single column layout
- Large touch targets for CTAs (44x44px minimum)
- Alt text for all images
- Plain text version provided

### Version Control (continued)

**Design System Versioning:**
- Major version (1.0, 2.0): Significant overhaul, breaking changes
- Minor version (1.1, 1.2): New components, non-breaking additions

Document all changes with date, description, and impact:
```
## Version 1.2.0 - June 2025
### Added
- New testimonial card component
- Celebration animation for course completion
- Dark mode color palette (experimental)

### Changed
- Updated button padding for better mobile touch targets
- Refined color contrast ratios for WCAG 2.2 compliance

### Fixed
- Modal close button not keyboard accessible
- Icon alignment inconsistency in navigation
```
